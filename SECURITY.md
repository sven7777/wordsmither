# Security Policy

## Scope

Wordsmither runs **entirely in your browser**. It has no backend, no API calls,
no accounts, and no data collection — the text you paste never leaves your
device. As a result the attack surface is small, limited to the static
client-side bundle.

## Reporting a vulnerability

If you believe you've found a security issue (for example, a way the app could
execute untrusted input or leak data off-device), please report it privately:

- Open a [private security advisory](https://github.com/sven7777/wordsmither/security/advisories/new), **or**
- Email **derek@slackersbrew.com**.

Please don't open a public issue for security reports. We'll acknowledge your
report and work on a fix; given the project's scope, expect a best-effort
timeline.
